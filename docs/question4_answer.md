# 问题四答案

最后更新：2026-04-23

本文直接对应原任务书中的问题四：

> 分析各机型使用情况，并提出是否需要进一步调整机队或航班计划的建议。

当前正式口径已经切换为 20 场景 Monte Carlo 主线。本回答基于当前主模型结果目录 [`results/runs/model_current`](../results/runs/model_current)，该目录现在保存的是 20 场景结果；原 5 场景结果已归档到 [`results/runs/model_current_5scenario_archive_2026-04-23`](../results/runs/model_current_5scenario_archive_2026-04-23)。统一交接说明见 [`monte_carlo_20_scenario_handoff.md`](monte_carlo_20_scenario_handoff.md)。

## 一、结论先行

20 场景结果下，网络并不是“总机队数量不够”，而是“少数主力机型明显偏紧、部分机型明显闲置”。因此，问题四的核心建议不是全面扩充整个机队，而是优先做结构性调整：

1. 优先围绕 `F16C0Y165` 和 `F0C0Y76` 两类主力机型做调配、换大机型和补容量。
2. 不建议优先扩充当前完全闲置的机型家族。
3. 对极低载客率且亏损的少数任务，应优先进入班次与时刻复核清单。
4. 对高载客率、高利润、正影子价格的航段，应优先考虑增容，而不是平均化地加飞机。

从管理含义上看，这意味着当前方案更像是“资源结构错配”，而不是“资源总量不足”。

## 二、现状判断

根据 [`scenario_summary.csv`](../results/runs/model_current/scenario_summary.csv)，当前主模型结果表现为：

- 总收入 `11,902,969.36`
- 总成本 `6,455,770.83`
- 总利润 `5,447,198.52`
- 总未满足需求 `13,539.85`
- 平均载客率 `85.83%`
- 加权载客率 `87.49%`
- 高影子价格航段数 `602`

这说明当前解总体盈利且载运率较高，但网络中仍有大量容量紧约束航段，因此问题四的重点应放在“哪些资源是核心瓶颈、哪些资源没有被有效使用”。

20 场景图形输出位于 [`results/figures/question4`](../results/figures/question4)，详细分析包位于 [`results/analysis/question4_mc20_2026-04-23`](../results/analysis/question4_mc20_2026-04-23)。

## 三、各机型使用情况分析

核心依据来自 [`fleet_summary.csv`](../results/runs/model_current/fleet_summary.csv)。

### 1. 主力机型高度集中

当前真正承担网络主力运力和利润贡献的是两类机型：

- `F16C0Y165`
  - 可用 `48` 架，实际使用 `42` 架
  - 利用率 `87.50%`
  - 加权载客率 `89.62%`
  - 归因利润 `3,008,368.00`
  - 正影子价格航段数 `260`

- `F0C0Y76`
  - 可用 `38` 架，实际使用 `38` 架
  - 利用率 `100.00%`
  - 加权载客率 `86.33%`
  - 归因利润 `1,693,450.62`
  - 正影子价格航段数 `295`

这两类机型合计归因利润为 `4,701,818.62`，占总方案利润的 `86.32%`；同时承载了 `92.19%` 的正影子价格航段。因此，它们不是普通机型，而是当前网络的核心瓶颈资源。

### 2. 次主力机型承担补充作用

- `F12C12Y48`
  - 利用率 `100.00%`
  - 加权载客率 `69.94%`
  - 利润 `227,077.91`

- `F0C0Y72`
  - 利用率 `100.00%`
  - 加权载客率 `72.45%`
  - 利润 `60,060.24`

这两类机型已被完全使用，但利润贡献和瓶颈承载能力显著低于前两类，更像是补充性运力，而不是决定网络利润水平的主力资源。

### 3. 多类机型明显闲置

以下机型当前没有承担实质性的主力作用：

- `F8C12Y126`：39 架可用，0 架使用
- `F12C30Y117`：44 架可用，0 架使用
- `F8C12Y99`：9 架可用，0 架使用
- `F12C0Y132`：8 架可用，0 架使用
- `F12C0Y112`：10 架可用，0 架使用

这些完全闲置机型合计对应 `110` 架可用飞机，而当前总可用飞机数为 `211`。也就是说，超过一半的可用飞机并未形成有效的主力承运能力。

因此，问题四的第一结论非常明确：

> 当前不存在“所有机型都不够”的问题，真正紧缺的是少数适配性很强的主力机型。

### 4. 闲置机型更像性价比问题，而不是求解失败

当前结果里确实有若干机型 `0` 使用，但这并不必然意味着模型没有找到最优解。更合理的解释是：在当前模型口径下，这些机型的单位运力成本偏高、综合性价比较低，因此没有进入最优分配方案。

从机队参考成本表粗看，当前主力机型恰好也是单位座位成本更有优势的机型：

- `F16C0Y165`：每座每小时成本约 `27.89`
- `F0C0Y76`：每座每小时成本约 `31.45`
- `F8C12Y126`：每座每小时成本约 `66.27`
- `F12C0Y132`：每座每小时成本约 `47.58`

这说明“机型闲置”当前更像是性价比筛选结果，而不是求解异常。

## 四、是否需要进一步调整机队

答案是：需要调整，但更适合做结构性调整，而不是全面扩编。

如果允许补充机队或重新分配运力，优先顺序应集中在：

- `F16C0Y165`
- `F0C0Y76`

原因是这两类机型已经同时表现出高利用率、高载客率、高利润贡献和高正影子价格航段覆盖。它们的边际容量价值最高，新增一架或从其他任务释放一架，都更可能带来真实的利润改善。

对 `F8C12Y126`、`F12C30Y117`、`F8C12Y99`、`F12C0Y132`、`F12C0Y112` 这类完全闲置机型，不建议优先扩充。当前模型已经给出足够强的信号，说明这些机型在现有网络中不是主力适配机型。

因此，如果要写成任务书语言，可以表述为：

> 进一步调整机队是必要的，但方向应是围绕高适配主力机型做强化，而不是平均化扩充所有机型。

## 五、是否需要进一步调整航班计划

答案也是需要，而且优先级比“总量扩机”更高。

根据 [`task_adjustment_candidates.csv`](../results/runs/model_current/task_adjustment_candidates.csv) 和 [`leg_adjustment_candidates.csv`](../results/runs/model_current/leg_adjustment_candidates.csv)：

- `473` 个任务被标记为 `Upgauge_or_AddCapacity`
- `602` 个航段被标记为 `Upgauge_or_AddCapacity`
- `93` 个任务被标记为 `Downgauge`
- `5` 个任务被标记为 `Review_Schedule`

### 1. 应优先增容的任务和航段

最值得优先增容的代表任务包括：

- `AA0123` `TFB -> GBJ`，利润 `37,294.80`
- `AA0004` `TFB -> EDB`，利润 `32,488.55`
- `AA0129` `TFB -> GBJ`，利润 `25,690.92`
- `AA0087` `TFB -> BOD`，利润 `25,486.23`
- `AA0083` `TFB -> BOD`，利润 `24,877.32`

典型的高价值瓶颈航段包括：

- `AA2701BODMBT`
- `AA2702IOMCMJ`
- `AA0004TFBEDB`
- `AA0123TFBGBJ`
- `AA0871IOMBOD`
- `AA0862PHHQEY`

这些对象共同具备三个特征：载客率高、影子价格为正、利润或价值贡献高。因此，它们是最适合优先增容、换大机型或研究补班的区域。

### 2. 应优先降级的任务

被标记为 `Downgauge` 的任务共有 `93` 个。代表例子包括：

- `BA2389` `ZZK -> TFB`，期望载客率 `18.79%`
- `AA0195` `BOD -> GBJ`，期望载客率 `20.21%`
- `AA0042` `CFU -> BOD`，期望载客率 `24.67%`
- `AA0106` `BOD -> TFB`，期望载客率 `26.70%`
- `BA2382` `TFB -> ZZK`，期望载客率 `27.70%`

这类任务更适合换小机型，以释放主力机型去承接高价值瓶颈任务。

### 3. 应进入复核清单的任务

当前有 `5` 个任务被标记为 `Review_Schedule`：

- `AA0138` `BOD -> PSE`，载客率 `9.34%`，利润 `-13,668.36`
- `BA2631` `TFB -> QEY`，载客率 `17.06%`，利润 `-1,072.70`
- `AA0082` `BOD -> TFB`，载客率 `49.67%`，利润 `-751.35`
- `BA2484` `TFB -> CMJ`，载客率 `21.46%`，利润 `-354.00`
- `BA2111` `QTD -> TFB`，载客率 `27.43%`，利润 `-4.90`

这些任务不一定要立刻取消，但至少应当进入班次、时刻和保留必要性复核名单。若缺少明显的网络连接价值，应优先调整甚至压缩。

## 六、航班计划调整的进一步证据

从 [`product_assignment_analysis.csv`](../results/runs/model_current/product_assignment_analysis.csv) 看，未满足需求并不是均匀分布的，而是集中在少数瓶颈航段上。

按瓶颈航段聚合后，未满足需求最高的航段包括：

- `AA0858PHHTFB`：`156.50`
- `AA0851TFBIOM`：`155.55`
- `AA0477MBYTFB`：`155.35`
- `AA0020TFBPSE`：`139.90`
- `AA0259QWSMBY`：`136.60`
- `AA0521CVSTFB`：`125.90`

这说明问题四不能只从“哪架飞机忙不忙”来回答，还必须指出：

> 当前未满足需求主要由少数瓶颈航段挤压形成，因此调整航班计划时，应围绕这些瓶颈点做定向增容。

## 七、面向任务书的最终回答

如果直接按题目作答，可以整理为如下表述：

1. 当前并购后机队使用明显分化，`F16C0Y165` 和 `F0C0Y76` 是网络的核心主力机型，利用率、载客率、利润贡献和瓶颈航段覆盖都显著高于其他机型。
2. 多个机型家族处于明显闲置状态，说明当前问题不是总机队规模不足，而是机型结构与网络需求之间存在错配。
3. 因此，有必要进一步调整机队，但不建议平均化扩充所有机型，而应优先围绕 `F16C0Y165` 和 `F0C0Y76` 做强化或释放。
4. 同时，有必要进一步调整航班计划。对 `TFB-BOD`、`TFB-GBJ`、`TFB-EDB`、`IOM-CMJ`、`BOD-MBT` 等高载客率、高价值、正影子价格航段，应优先考虑增容或换大机型。
5. 对低载客率但仍占用运力的任务，应优先考虑降级机型；对极低载客率且亏损的少数任务，应进入班次和时刻复核清单。

因此，问题四的最终结论是：

> 需要进一步调整机队与航班计划，但调整重点应是“结构优化”和“瓶颈增容”，而不是简单扩大总运力。

20 场景主线相对原 5 场景仅 `54/739` 个任务发生机型调整，总利润变化约 `-0.32%`，说明上述结构性调整建议具有较好的稳定性。
